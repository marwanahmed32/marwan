سارة العقارات — وكيل أتمتة التسويق والرد العقاري الذكي

نظام متكامل يحل مشكلة ضياع العملاء المهتمين بسبب بطء رد موظفي المبيعات، حيث يعمل الوكيل على مدار 24 ساعة للترحيب وتصنيف والرد على استفسارات العقارات بكفاءة بشرية.

الهدف الأساسي: تقليل وقت الرد لـ 0 ثانية وتأهيل العملاء
مدة التطوير: 3 أسابيع
المنصات: WhatsApp Business, Telegram, Facebook Messenger
محرك العمليات: n8n Workflow Automation

كيف يعمل النظام؟ (خطوات التنفيذ التقنية)

  1. استقبال الحدث (Webhook Trigger):

    عندما يرسل العميل رسالة جديدة على منصات التواصل، يقوم خادم Webhook مدمج في n8n بالتقاط الرسالة وبيانات المرسل فوراً وتمريرها لنظام التصفية.

  2. تحليل النية والذكاء الدلالي (Intent Parsing):

    يتم إرسال نص الرسالة إلى نموذج GPT-4o عبر OpenAI API باستخدام نظام توجيه (System Prompt) معد خصيصاً لاستخلاص 4 قيم رئيسية: (الغرض: بيع/شراء/إيجار، نوع العقار، الميزانية المتوقعة، والمنطقة الجغرافية).

  3. البحث والمطابقة الذكية (Database Retrieval):

    إذا كانت هناك نية للشراء أو الإيجار، يقوم النظام بالاستعلام التلقائي من قاعدة بيانات العقارات. يتم البحث باستخدام البحث النصي والدلالي للحصول على أفضل 3 عقارات مطابقة تماماً لرغبة العميل.

  4. توليد العرض المناسب (Content Generation):

    تتم إعادة صياغة بيانات العقارات المطابقة بأسلوب تسويقي جذاب ومكتوب بلغة احترافية تتناسب مع طبيعة العميل، مع توفير روابط للصور والخرائط.

  5. المتابعة الآلية (Follow-up Loop):

    يتم إضافة العميل في قائمة متابعة مؤتمتة (بعد 24 ساعة) للتأكد من ملاءمة المقترح وإرسال استطلاع رأي أو حجز موعد مكالمة مع الخبير العقاري البشري.

البوت المحاسبي — أتمتة استخراج الفواتير والترحيل المالي

تخلص من الإدخال اليدوي المتكرر للفواتير الورقية والإلكترونية. يقوم هذا البوت بقراءة صور الفواتير، استخراج البنود والضرائب وتخزينها في الدفاتر المالية تلقائياً من خلال محادثة تليجرام بسيطة.

الهدف الأساسي: تحويل الفواتير الورقية لبيانات منظمة في ثوانٍ
مدة التطوير: شهران (تطوير واختبار)
المنصات: Telegram Bot API, Google Sheets, Accounting ERP
التقنيات المستخدمة: Node-RED / n8n, OpenAI Vision API, Javascript

كيف يعمل النظام؟ (خطوات التنفيذ التقنية)

  1. قناة الإدخال (Telegram Channel):

    يقوم المحاسب أو السائق أو الموظف بالتقاط صورة للفاتورة الورقية وإرسالها لبوت التليجرام الخاص بالشركة أثناء تنقله.

  2. معالجة الصور والـ OCR الذكي:

    يتم سحب الصورة عبر نفق آمن باستخدام Telegram Bot API وتمريرها لنظام OCR معزز بـ Vision LLM لقراءة الأرقام والنصوص في الإضاءات المختلفة والتصوير المائل.

  3. هيكلة البيانات المالية (JSON Parsing):

    يقوم نموذج الذكاء الاصطناعي باستخراج حقول محددة: (اسم المورد، الرقم الضريبي، تاريخ الفاتورة، التفاصيل، المبلغ الأساسي، نسبة القيمة المضافة، وإجمالي المبلغ) بصيغة JSON منظمة.

  4. التحقق والتصنيف المحاسبي:

    يقارن البوت قيمة الضريبة المحسوبة رياضياً بالضريبة المسجلة بالفاتورة للتأكد من سلامة الحسابات، ثم يصنف الفاتورة تلقائياً (مثل: نفقات ضيافة، وقود، إيجار).

  5. الترحيل والمزامنة (Data Sync):

    يتم كتابة البيانات المحققة مباشرة في السجل المالي على جداول Google Sheets، مع إرسال إشعار للمدير المالي على القناة الرسمية لتأكيد الصرف أو المراجعة.

المساعد القانوني الذكي — RAG Agent لتحليل العقود والقوانين

مساعد تقني يدعم الإدارات القانونية في الشركات بمراجعة العقود الطويلة وتحديد الثغرات القانونية الخطرة في ثوانٍ، بدلاً من قضاء ساعات طويلة في فحص الوثائق التقليدية.

الهدف الأساسي: استخلاص الالتزامات والثغرات وصياغة المذكرات
مدة التطوير: 4 أسابيع
قاعدة البيانات: Pinecone Vector Database (RAG)
النماذج الأساسية: Claude 3.5 Sonnet / GPT-4o

كيف يعمل النظام؟ (خطوات التنفيذ التقنية)

  1. استقبال المستندات وتقسيمها (Document Chunking):

    يتم رفع العقود بصيغة PDF أو Word، ويقوم كود Python مكتوب بـ LangChain بتقسيم المستند إلى فقرات صغيرة متداخلة لضمان عدم ضياع السياق في البنود الطويلة.

  2. توليد المتجهات (Vector Embeddings):

    يتم تحويل كل فقرة إلى تمثيل رياضي (Embedding) باستخدام نموذج OpenAI text-embedding-3، ثم حفظها في قاعدة بيانات Pinecone.

  3. الاسترجاع المعزز بالسياق (Semantic Retrieval):

    عندما يطرح المستخدم سؤالاً مثل "ما هي شروط فسخ العقد والغرامات؟"، يقوم النظام بتحويل السؤال لمتجه والبحث دلالياً في Pinecone لجلب الفقرات وثيقة الصلة فقط.

  4. تحليل واستخراج الثغرات (LLM Reasoning):

    يتم تمرير الفقرات المسترجعة إلى Claude 3.5 Sonnet مع موجه (Prompt) قانوني يطلب منه فحص الفقرات بدقة ومقارنتها بمواد القانون المدني المصري لكشف أي غبن أو بنود غير متوازنة.

  5. صياغة التقرير النهائي (Automated Briefing):

    توليد مذكرة ملخصة بصيغة ملف Word منظم تحتوي على: ملخص الالتزامات، جدول بالتواريخ الهامة، الثغرات المكتشفة، والتعديلات المقترحة للبنود.

  • كشف ثغرات وبنود جزائية مخفية في عقود توريد حقيقية وتنبيه أصحاب العمل لها.
